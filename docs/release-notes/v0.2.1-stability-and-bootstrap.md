# PaperWeave v0.2.1

Maintenance release focused on project bootstrap, CLI reliability, incremental correctness, and biomedical workflow stability.

## Highlights

- `init` now bootstraps a fresh project directory with default config, prompt templates, and `.env.example`
- `doctor` now reports missing config and prompt files without crashing
- non-LLM commands such as `ingest`, `export`, `citations`, and parsing no longer fail just because a prompt file is missing
- summary and QA task selection now retry previously failed papers during normal incremental runs
- summary cache invalidation now respects rendered prompt content, including `research_context`
- biomedical summary export now renders the biomedical schema instead of producing near-empty Markdown
- citation tracking now deduplicates citing paper stubs by multiple identifiers, not DOI alone
- ingest no longer crashes when a tracked file changes into content already represented by another tracked PDF

## User-Visible Improvements

- first-time setup is simpler and matches the documented quick start more closely
- `paperweave` is now the documented primary CLI command, while `paperctl` remains as a compatibility alias
- biomedical QA export includes clearer labels for methodology and clinical sections
- parsing degrades cleanly when PubMed lookup fails during biomedical pre-enrichment

## Reliability Fixes

- decoupled prompt validation from commands that do not require LLM prompts
- hardened citation stub upsert logic for no-DOI results
- improved incremental task reuse and re-run behavior
- extended regression coverage across CLI, parsing, export, citation tracking, and LLM flows
