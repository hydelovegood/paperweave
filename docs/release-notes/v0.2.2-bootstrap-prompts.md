# PaperWeave v0.2.2

Patch release focused on first-run usability after `init`.

## Fixes

- `paperweave init` now bootstraps production-ready prompt templates instead of placeholder prompts
- freshly initialized projects can move directly into `summarize` and `qa` workflows with the same structured prompt contracts used by the repository defaults
- added regression coverage to ensure bootstrapped prompt files contain the expected summary, QA, and biomedical schemas

## Why This Release Exists

v0.2.1 made fresh-project bootstrap work, but the generated prompt files were minimal placeholders rather than the actual structured templates expected by runtime workflows.

That meant a brand-new project could initialize successfully but still have a degraded first-run LLM experience.

v0.2.2 closes that gap.
