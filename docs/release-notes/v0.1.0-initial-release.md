# PaperWeave v0.1.0

Initial public release of PaperWeave（溯源文库）, a local-first CLI for building a structured paper library from PDFs.

## Highlights

- Explicit-path PDF ingestion with SHA256 deduplication
- Unified parsing pipeline with `DeepXiv` first and `PyMuPDF` fallback
- Structured summary generation
- Structured reviewer / interview / author-defense QA generation
- Forward citation tracking with OpenAlex, Semantic Scholar, Crossref, and Unpaywall
- Markdown export for `summary.md` and `QA.md`
- `doctor` command for local environment and LLM connectivity checks

## What It Is Good For

- Personal research paper libraries
- Literature review workflows
- Revisiting old papers with structured summaries and deeper questions
- Tracking which recent papers cite a designated classic paper

## Current Boundaries

- Single-user local CLI workflow
- SQLite backend
- No GUI
- No background worker / job queue
- Forward-citation links are tracked, but automatic PDF collection is still limited

## Tested Workflow

Validated on a real local paper folder with:

- project initialization
- PDF ingestion
- parsing
- summary generation
- QA generation
- citation tracking
- Markdown export

## Notes

- LLM raw outputs may be stored in `data/logs/llm/`
- API keys are loaded from `.env`
- `DeepXiv` requires a valid token for arXiv-structured parsing
